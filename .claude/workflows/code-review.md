# /code-review

按以下步骤审查代码变更:

1. 读当前 diff（`git diff main...HEAD`，如果无 main 则用 `master`）
2. 按以下维度分析:
   - 正确性:逻辑是否有 bug?边界 case 是否处理?
   - 安全性:输入校验、注入风险、密钥泄露?
   - 性能:不必要的 O(n²)、冗余请求、未释放资源?
   - 可维护性:命名清晰、函数够短、有必要的注释?
3. 每个发现标注严重程度:
   - 🔴 高 — 会出 bug / 安全漏洞 / 数据丢失
   - 🟡 中 — 性能问题 / 潜在风险 / 技术债
   - 🟢 低 — 命名建议 / 注释补充 / 代码风格
4. 输出审查报告到 `review.md`，按严重程度排序
